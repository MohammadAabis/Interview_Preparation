1- What is HTML?
Ans - Foundation of any web page, it gives the structure of your website by allowing adding contents and buttons and many more things.

2- <!DOCTYPE html> means?
Ans - It tell the browser to clarify the version of html and how to interpret the code. In this, it is HTML5.

3- What are Meta Tags?
Ans - It is used to provide additional information about a web page.
--> charset="UTF-8" -- It is important and used to display different language characters on browser correctly.
--> name="viewport" -- It is used for responsive design and also very important.
--> name="description" -- it is used for SEO purpose to search this web page.

4- Difference between <div> and <span> element?
Ans - Div is used to group and structure the content on a web page. On the other hand, Span is used to apply styles for specific section inside a div.

5- Semantic Elements?
Ans - Those Elements which gives you a complete sense of data they contain like <header>, <footer>, <section>, <address> and <main>.

6- Difference between Block-level and Inline Elements?
Ans - Block elements (<div>,<p>,<h1>,<ul>,<li>,<table>) create new lines while inline elements (<span>,<a>,<strong>,<emp>,<img>,<input>) do not start new line.
--> You can set both height and width for block element for inline element you cant because it only take it's content length and protect it from overlapping. Horizontal padding and margins can be applied to inline elements 
When you put inline element inside block element and give flex to parent then the child behaves as block element due to flex property. If you don't give flex it remains inline. You can't put block element inside inline element because it gives error or warning.

7- Absolute and Relative URLs?
Ans - Absolute URLs provide full URLs like (http://www.test.com) and take you to another websites. On the other hand relative URLs provide URLs like (page.html or images/image1.png) and browser behave it differently and search it inside the website.

8- HTML best practices for SEO?
Ans - Use Semantic HTML Elements, Optimize page titles, use Meta description tags, use proper heading tags, optimize image by alt attributes.

9- Tags, Elements and Attributes?
Ans - <input type="text"></input> in this the simple <input> is called tags, and inside it there is attribute which is type and when we use starting tag and ending tag it is called html elements.

10- <b>, <strong>, <i>, <emp>?
Ans - b makes the text bold and i for italic. Strong makes text bold and tell the browser it is important and emp is used to emphasize the text while reading.

11- Data Attributes?
Ans - It provide a way to add a custom data attributes to add additional information.

12- HTML vs HTML5 ?

HTML:
--> Doctype declaration is too long and complicated.
--> Div tag was used all the time.
--> It store data on a user's browser was through cookies, which were very limited in size and were sent with every HTTP request, slowing down performance

HTML5:
--> Doctype declaration is easy.
--> It introduce semantic element like <header>, <footer>.
--> It introduced localStorage and session Storage. These provide much larger storage capacity and are more secure, as the data is not sent with every HTTP request.

13- How HTML and JSX compile ?

When JSX is compiled in JS through babel compiler, there is no direct "HTML compilation". React and ReactDOM are responsible for rendering those objects into real DOM nodes.
